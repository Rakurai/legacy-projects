# Implementation Plan: V1 Known Issue Fixes

**Branch**: `002-v1-issue-fixes` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)

---

## Summary

Fix two build-pipeline/server bugs found in the V1 usability test:

1. **I-001**: Declaration/definition split entities are stored as two separate database rows, fragmenting graph metrics from documentation. Fix: deduplicate during `merge_entities()` using a new early-stage GML node ID lookup, keeping the graph-referenced compound and merging doc fields from the other.
2. **I-002**: Per-query score normalization makes all top scores 1.0, preventing noise filtering. Fix: normalize keyword scores intra-query, replace normalization with a combined-score threshold (~0.2).
3. **I-004**: Verification that `stc` (fan_in=640) becomes `is_contract_seed=true` after I-001 — retained as a named regression test.

## Phase 1: Design

### I-001 Implementation Detail

**`graph_loader.py`**: Add `load_graph_node_ids(artifacts_path: Path) -> frozenset[str]`

Reads the raw GML file via NetworkX, returns the node ID set. No entity lookup. Runs before `merge_entities()`.

```
load_graph_node_ids(artifacts_path: Path) -> frozenset[str]
    G = nx.read_gml(gml_path)
    return frozenset(G.nodes())
```

**`entity_processor.py`**: Modify `merge_entities()`

New signature: `merge_entities(entity_db, doc_db, graph_node_ids: frozenset[str]) -> list[MergedEntity]`

Logic:
1. For each entity in `entity_db.entities`, collect `(dedup_key, entity, doc)` tuples.
2. Group by `dedup_key = (entity.name, entity.signature, entity.location.file)`.
3. For single-entity groups: construct `MergedEntity` unchanged.
4. For multi-entity groups:
   - Find survivor: entity whose `entity.id.member or entity.id.compound` is in `graph_node_ids`.
   - If neither: `raise BuildError(f"entity '{entity.name}' has no graph-referenced compound")`.
   - If both: keep the one whose file path is a definition file; if ambiguous, prefer the one with non-null doc.
   - Merge doc: survivor's doc = survivor_doc or sibling_doc.
   - Log each merge.
5. Return deduplicated list.

**`build_mcp_db.py`**: Insert `load_graph_node_ids()` call before `merge_entities()`.

No other pipeline changes. `load_graph_edges()` call site is unchanged.

### I-002 Implementation Detail

**`server/search.py` — `_merge_scores()`**:

Before applying `_KEYWORD_WEIGHT`, normalize `keyword_scores` within the result set:

```python
if keyword_scores:
    max_kw = max(keyword_scores.values())
    keyword_scores = {eid: s / max_kw for eid, s in keyword_scores.items()}
```

**`server/search.py` — `hybrid_search()` and `hybrid_search_usages()`**:

Replace the per-query normalization block:
```python
# REMOVE:
max_score = ranked[0][1] if ranked else 1.0
normalizer = max_score if max_score > 0 else 1.0
score = min(score / normalizer, 1.0)

# REPLACE WITH:
_SCORE_THRESHOLD = 0.2
results = [r for r in results if r.score >= _SCORE_THRESHOLD]
# (exact matches already score ≥ 10.0, so they pass unconditionally)
```

Constant `_SCORE_THRESHOLD = 0.2` defined at module level. Value validated against SC-003/SC-004 before commit. If empirical validation requires adjustment, update the constant.
